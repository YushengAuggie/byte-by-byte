## 场景 / Scenario

你在做一个 dashboard，需要在多个页面复用“拉取数据 + loading/error + 取消请求 + 刷新”这套逻辑：
- 订单列表页 / Orders page
- 用户列表页 / Users page
- 某个详情页 / Details page

如果你把这套逻辑复制粘贴到每个组件里，你会得到：重复代码、bug 修复要改 N 份、依赖数组容易写错。

You're building a dashboard and need the same "fetch data + loading/error + abort + refresh" logic across multiple pages. Copy/paste leads to duplication and subtle useEffect bugs.

---

## 核心概念 / Core idea

**自定义 Hook = 把可复用的状态 + 副作用封装成一个函数（以 use 开头）**。
它不是“为了抽象而抽象”，而是为了：
1) 减少重复
2) 固化正确姿势（cleanup、依赖、边界）
3) 让组件回到“渲染 UI”本职

A custom hook packages reusable state + effects into a function (must start with `use`). It reduces duplication, bakes in best practices, and keeps components focused on UI.

## 代码示例 / Code snippet (production-ready-ish)

下面这个 `useAsync` 展示：
- 统一处理 loading/error
- 支持手动 refresh
- 支持 AbortController 防止卸载后 setState

```tsx
import { useCallback, useEffect, useMemo, useState } from "react";

type AsyncState<T> = {
  data: T | null;
  error: Error | null;
  loading: boolean;
  refresh: () => void;
};

export function useAsync<T>(
  fn: (signal: AbortSignal) => Promise<T>,
  deps: unknown[]
): AsyncState<T> {
  const [data, setData] = useState<T | null>(null);
  const [error, setError] = useState<Error | null>(null);
  const [loading, setLoading] = useState(false);
  const [nonce, setNonce] = useState(0);

  const refresh = useCallback(() => setNonce((x) => x + 1), []);

  const run = useCallback(async (signal: AbortSignal) => {
    setLoading(true);
    setError(null);
    try {
      const result = await fn(signal);
      if (!signal.aborted) setData(result);
    } catch (e) {
      if (!signal.aborted) setError(e as Error);
    } finally {
      if (!signal.aborted) setLoading(false);
    }
  }, [fn]);

  useEffect(() => {
    const controller = new AbortController();
    run(controller.signal);
    return () => controller.abort();
    // eslint-disable-next-line react-hooks/exhaustive-deps
  }, [...deps, nonce]);

  return useMemo(() => ({ data, error, loading, refresh }), [data, error, loading, refresh]);
}
```

Usage:
```tsx
const state = useAsync(async (signal) => {
  const res = await fetch("/api/orders", { signal });
  if (!res.ok) throw new Error("Failed to load orders");
  return res.json() as Promise<Order[]>;
}, []);
```

## 猜猜这段代码输出什么？/ Quiz: what does this log?

```tsx
function Demo() {
  const [n, setN] = useState(0);

  useEffect(() => {
    console.log("effect", n);
    return () => console.log("cleanup", n);
  }, [n]);

  return (
    <button onClick={() => setN((x) => x + 1)}>+1</button>
  );
}
```

A. 点击一次：`effect 0` 然后 `effect 1`（没有 cleanup）
B. 点击一次：`cleanup 0` 然后 `effect 1`
C. 点击一次：`effect 0` 然后 `cleanup 1` 再 `effect 1`
D. 点击一次：只打印 `cleanup 0`

**正确答案 / Correct: B**（依赖变化时会先执行上一次 effect 的 cleanup，再执行新的 effect）

## 常见错误 → 正确做法 / Common mistake → Correct approach

❌ **错误：自定义 Hook 里 conditionally 调用 hooks**
```tsx
function useMaybeThing(flag: boolean) {
  if (flag) {
    useEffect(() => {
      // ...
    }, []);
  }
}
```

✅ **正确：Hook 调用顺序固定，把条件放到 effect 内部**
```tsx
function useMaybeThing(flag: boolean) {
  useEffect(() => {
    if (!flag) return;
    // ...
  }, [flag]);
}
```

❌ **错误：返回不稳定的对象，导致下游组件频繁 re-render**
```tsx
return { data, loading, error, refresh };
```

✅ **正确：useMemo 保持引用稳定（必要时）**
```tsx
return useMemo(() => ({ data, loading, error, refresh }), [data, loading, error, refresh]);
```

## 什么时候用 / 什么时候别用 / When to use / when NOT to

✅ 适合用自定义 Hook：
- 逻辑跨多个组件重复出现（数据拉取、表单校验、键盘快捷键、权限判断）
- 需要封装“正确姿势”（cleanup、订阅/退订、事件监听）
- 想把组件从“逻辑泥潭”中解放出来

❌ 不适合：
- 只在一个组件用一次、且很短的逻辑（过度抽象会降低可读性）
- 你还没搞清楚依赖/生命周期（先写清楚，再抽）

Use custom hooks for repeated cross-component logic and to encapsulate best practices. Avoid for one-off tiny logic or when abstraction would hide clarity.

## 🧒 ELI5
把自定义 Hook 想成“你自己做的乐高小模块”。你先把常用的零件拼成一块（loading/error/refresh），以后每次需要就直接拿来用，不用每次从零开始拼。

A custom hook is like a LEGO module you built once. Next time you need it, you reuse the module instead of rebuilding from scratch.
