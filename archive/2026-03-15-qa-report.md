## ⚠️ Detail: Frontend `content` property usage

In the flex gotcha section:
```css
.a { flex: 1 1 auto; content: "longer text"; }
```
`content` is not a valid CSS property on regular HTML elements (it's for `::before`/`::after` pseudo-elements). This won't actually demonstrate the difference in a browser — readers who copy-paste would be confused. 

The conceptual explanation is 100% correct; the code example just needs to use actual HTML inner text or `width` differences to illustrate the point.

**Suggested fix:** Replace with a note like "if .a has longer text content in HTML, it will receive more space."
